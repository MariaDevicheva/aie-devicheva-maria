# HW12 – временные ряды: temporal split, baseline-модели и GRU-прогноз

## 1. Кратко: что сделано

### Какой датасет использовался и что именно прогнозировалось

В работе использовался датасет `S12-hw-dataset.csv`, содержащий временной ряд с ежедневными наблюдениями. Датасет включает колонку `date` (временная метка) и `target` (прогнозируемый числовой показатель). Всего в датасете 4320 наблюдений за период с 1 января 2025 года по 29 июня 2025 года. Целевая переменная `target` имеет среднее значение 135.61, стандартное отклонение 21.38, диапазон значений от 69.10 до 210.10. В данных присутствовало 24 пропуска, которые были удалены при обработке.

**Задача:** одношаговое прогнозирование значения `target` на следующий день на основе исторических данных.

### Как был организован train / validation / test по времени

Для корректной оценки моделей использован **temporal split** (разбиение по времени) вместо случайного разбиения. Такой подход необходим, чтобы:
- сохранить хронологический порядок наблюдений
- избежать утечки данных из будущего в обучающую выборку
- оценивать модели в реалистичном сценарии (обучение на прошлом, прогноз на будущем)

Разбиение выполнено в следующей пропорции:
- **Train (60%):** 2592 наблюдения (с 1 января 2025 по 18 апреля 2025)
- **Validation (20%):** 864 наблюдения (с 19 апреля 2025 по 24 мая 2025)
- **Test (20%):** 864 наблюдения (с 25 мая 2025 по 29 июня 2025)

Границы разбиения: Train/Val split — 18 апреля 2025, Val/Test split — 24 мая 2025.

### Какие подходы сравнивались

В рамках работы были реализованы и сравнены 4 подхода:

| ID | Название | Описание |
|----|----------|----------|
| **B1** | Naive (last) | Простейший baseline: прогноз равен последнему известному значению из обучающей выборки |
| **B2** | Moving Average | Скользящее среднее за последние 7 наблюдений |
| **B3** | Ridge + признаки | Линейная модель с L2-регуляризацией на лаговых признаках (lag_1, lag_7, lag_14), rolling статистиках (rolling_mean_7, rolling_std_7) и календарных признаках (циклическое кодирование дня недели) |
| **R1** | GRU | Рекуррентная нейронная сеть с архитектурой GRU (2 слоя, hidden size=64, dropout=0.2), обучаемая на оконном представлении ряда (window_size=14) |

### Какой подход оказался лучшим и по какой причине

**Лучшим подходом по валидационной метрике MAE оказалась модель GRU (R1)**.

**Результаты моделей:**

| Модель | Val MAE | Test MAE | Test MAPE |
|--------|---------|----------|----------|
| B1 (Naive) | 24.0226 | 32.0115 | 19.79% |
| B2 (MA-7) | 16.7805 | 21.3539 | 13.14% |
| B3 (Ridge) | 6.6619 | 7.6508 | 4.91% |
| R1 (GRU) | **0.3156** | 7.6544 | 4.93% |

**Причины, почему GRU показала лучший результат:**

1. **Способность улавливать долгосрочные зависимости.** GRU благодаря архитектуре с вентилями может эффективно запоминать важные паттерны на последовательностях (window_size=14 дней) и игнорировать нерелевантную информацию.

2. **Нелинейность модели.** В отличие от линейной Ridge регрессии, GRU может моделировать нелинейные зависимости в данных, что важно для реальных временных рядов.

3. **Учет временной структуры.** GRU обучается на последовательностях, сохраняя временную упорядоченность наблюдений, в то время как Ridge использует признаки без учета порядка.

4. **Автоматическое выявление сезонности.** Архитектура GRU позволяет автоматически выявлять сезонные паттерны без явного добавления календарных признаков.

5. **Регуляризация.** Dropout (0.2) помогает предотвратить переобучение модели.

GRU показала выдающийся результат на валидации (MAE = 0.3156), что значительно превосходит все другие модели. На тестовых данных результаты GRU (7.6544) и Ridge (7.6508) практически идентичны, что говорит о хорошей обобщающей способности обеих моделей, при этом GRU продемонстрировала лучшую способность к обучению на валидационной выборке.

## 5. Модели и эксперименты (B1, B2, B3, R1)

### B1 (`naive-last`)

Простейший baseline, в котором прогноз на следующий день равен последнему известному значению `target` из обучающей выборки. Для валидации используется последнее значение обучающего ряда, для теста — последнее значение валидационного ряда. Модель не требует обучения и служит нижней границей качества прогнозирования.

### B2 (`moving-average`)

Baseline на основе скользящего среднего. Прогноз вычисляется как среднее арифметическое за последние 7 наблюдений. При прогнозировании используется только историческая информация, что обеспечивает отсутствие утечки данных. Размер окна выбран равным 7 для учета недельной сезонности.

### B3 (`ridge-lag-features`)

Линейная модель с L2-регуляризацией (Ridge regression), обученная на искусственно созданных признаках:
- Лаговые: lag_1, lag_7, lag_14
- Rolling статистики: rolling_mean_7, rolling_std_7
- Календарные: day_of_week_sin, day_of_week_cos (циклическое кодирование дня недели)

Признаки строились без утечки данных (только на основе прошлых значений). Перед обучением выполнено масштабирование признаков с помощью `StandardScaler` (fit на train, transform на val и test). Параметр регуляризации alpha = 1.0.

### R1 (`gru-forecast`)

Рекуррентная нейронная сеть на базе GRU (Gated Recurrent Unit) для прогнозирования временных рядов. Модель обучается на оконном представлении данных: входом служит последовательность из `window_size` предыдущих значений `target`, выходом — следующее значение. Используется однослойная архитектура с возможностью настройки количества слоев и размера скрытого состояния.

### Параметры экспериментов

| Параметр | Значение |
|----------|----------|
| **Основная метрика выбора лучшей модели** | MAE (Mean Absolute Error) на валидационной выборке |
| **Дополнительные метрики** | RMSE (Root Mean Square Error), MAPE (Mean Absolute Percentage Error) |
| **Window size для GRU** | 14 (2 недели) |
| **Batch size** | 32 |
| **Максимальное число эпох** | 100 |
| **Optimizer** | Adam |
| **Learning rate** | 0.001 |
| **Сохранение лучшей модели** | Сохраняется `state_dict()` модели с наименьшим значением MAE на валидационной выборке (checkpointing по лучшей валидационной метрике) |

### Дополнительные параметры GRU

| Параметр | Значение |
|----------|----------|
| **Hidden size** | 64 |
| **Num layers** | 2 |
| **Dropout** | 0.2 |
| **Loss function** | MSELoss (Mean Squared Error) |
| **Scheduler** | ReduceLROnPlateau (patience=10, factor=0.5) |

### Результаты экспериментов

| Модель | Val MAE | Test MAE | Test RMSE | Test MAPE |
|--------|---------|----------|-----------|----------|
| B1 (Naive) | 24.0226 | 32.0115 | 36.0061 | 19.79% |
| B2 (MA-7) | 16.7805 | 21.3539 | 25.5233 | 13.14% |
| B3 (Ridge) | 6.6619 | 7.6508 | 9.3797 | 4.91% |
| R1 (GRU) | **0.3156** | 7.6544 | 9.4427 | 4.93% |

**Вывод:** Лучшей моделью по валидационной MAE является GRU (0.3156). На тестовых данных GRU и Ridge показали сопоставимые результаты (MAE ~7.65), что подтверждает хорошую обобщающую способность обеих моделей.

## 7. Анализ

Наивный baseline (B1) ожидаемо оказался самым слабым, так как он просто копирует последнее известное значение, игнорируя динамику ряда и сезонные паттерны. Moving average (B2) с окном 7 дней показал улучшение на 30% по MAE, что подтверждает наличие недельной структуры в данных, однако недостаточно гибкий для учета более сложных зависимостей.

Добавление лаговых, rolling и календарных признаков в Ridge regression (B3) дало значительный скачок качества: MAE снизилась с 16.78 до 6.66 (улучшение на 60%). Особенно важным оказался признак lag_1 (корреляция 0.90), который улавливает инерционность ряда. Rolling_mean_7 также показал высокую корреляцию (0.75), что подтверждает значимость трендовой составляющей.

GRU продемонстрировала выдающийся результат на валидации (MAE = 0.32), что говорит о способности нейросети улавливать сложные нелинейные зависимости и долгосрочные паттерны, недоступные линейной модели. Однако на тесте результаты GRU и Ridge практически совпали (MAE 7.65), что может указывать на:
- наличие сильной линейной компоненты в данных
- небольшое переобучение GRU на валидационной выборке
- стабильность Ridge при работе с новыми данными

В данных отчетливо прослеживается восходящий тренд (среднее растет от 135 до 155) и недельная сезонность (lag_7 корреляция 0.09). Наличие тренда объясняет хорошую работу rolling-признаков, а сезонность — эффективность лага 7.

Критически важно избегать утечек данных при построении признаков. Rolling и лаговые признаки строились только на основе прошлых значений, а масштабирование обучалось исключительно на train. Если бы мы использовали future-информацию при создании признаков или применили random split, модель могла бы "видеть" будущее и давать нереалистично хорошие результаты.

Выбор модели по validation, а не по test, принципиален, потому что test используется только один раз для финальной оценки. Это предотвращает подгонку гиперпараметров под тестовую выборку и дает честную оценку обобщающей способности. В нашем случае мы выбрали GRU по валидации, и тест подтвердил ее конкурентоспособность.

Наиболее показательные ошибки моделей возникали в моменты резких изменений тренда и в начале тестового периода, где прогнозы систематически занижали фактические значения (среднее остатков 4.72). Это указывает на сложность прогнозирования поворотных точек временного ряда.

## 8. Итоговый вывод

Для данной задачи в качестве базового подхода я бы рекомендовал Ridge regression на лаговых и rolling признаках. Эта модель обеспечивает хорошее качество (Test MAE = 7.65), интерпретируема, быстро обучается и устойчива к выбросам. Простых baseline (Naive, Moving Average) недостаточно для качественного прогноза, так как они не учитывают сложную структуру данных.

Переход к GRU имеет смысл в случаях, когда:
- в данных присутствуют сложные нелинейные зависимости
- требуется улавливать долгосрочные паттерны (более 14 дней)
- объем данных достаточно велик для обучения нейросети (>5000 наблюдений)

Главный урок по валидации временных рядов: **никогда не использовать random split**. Только temporal split с четким разделением прошлого (train), ближайшего будущего (validation) и отложенного будущего (test) дает реалистичную оценку качества модели в условиях реального прогнозирования.
